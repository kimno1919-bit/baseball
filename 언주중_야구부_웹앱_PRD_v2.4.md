## 1. 상벌점 단계별 제재 규정

### 1-1. 단계 정의 (사용자 확정안)

| 단계 코드 | 합계 점수 | 상태 | 라인업 영향 |
|---|---|---|---|
| `NORMAL` | ≥ 0 | 정상 | 제한 없음 |
| `CAUTION` | −1 ~ −14 | 주의 | 라인업 등록 시 경고 뱃지만 표시 |
| `SUSPEND_1A` | −15 ~ −24 | 1경기 출전 정지 | 다음 1경기 라인업 등록 차단 |
| `SUSPEND_1B` | −25 ~ −39 | 1경기 출전 정지 | 다음 1경기 라인업 등록 차단 |
| `EXPELLED` | ≤ −40 | 퇴출 | 모든 경기 라인업 등록 차단, 계정 비활성화 |

> ⚠️ 확인 필요: `−15`와 `−25`가 모두 "1경기 출전 정지"로 표기되어 있습니다. 의도하신 대로 작성했으나, 오타(예: `−25`는 2경기)일 가능성이 있어 오픈 이슈 O-13에 기록했습니다.

### 1-2. 단계 진입·해제 규칙

#### NORMAL ↔ CAUTION
- 자동 전이, 카운터 없음
- 화면 표시만 변경

#### CAUTION → SUSPEND_1A (합계 −15 도달 시)
- 시스템이 `suspension_count = 1` 자동 부여
- 다음 경기 등록 시 라인업 등록 불가
- 경기 1회 종료(미출전) → `suspension_count = 0` → CAUTION으로 복귀
- 추가 벌점으로 −25 미만으로 떨어지면 SUSPEND_1B로 단계 상승

#### SUSPEND_1A → SUSPEND_1B (합계 −25 도달 시)
- `suspension_count = 1` 추가 부여 (잔여 정지 경기가 있을 경우 합산)
- 동일하게 1경기 미출전으로 소진

#### → EXPELLED (합계 −40 도달 시)
- 즉시 계정 상태 `EXPELLED`로 변경
- 라인업 등록 영구 차단
- 마이페이지 외 기능 비활성화 (출결 응답·기록 조회 모두 불가)
- 관리자만 수동 복권 가능 (사유 입력 필수)

#### 단계 하향 (상점 획득으로 합계 개선)
- 상점 획득으로 합계가 상위 단계 구간으로 진입하면 **즉시 단계 하향**
- 단, **이미 소진되지 않은 `suspension_count`는 유지** (출전 정지는 즉시 면제되지 않음)
  - 예: 합계 −15 → SUSPEND_1A + 정지 1경기 누적 → 상점 +20으로 합계 +5 → 단계는 NORMAL이지만 정지 1경기는 그대로 남아 다음 경기 미출전 후 소진
- EXPELLED는 합계 개선만으로 자동 해제되지 않음 (관리자 수동 복권만)

### 1-3. 라인업 등록(A07) 통합 동작

```
[관리자가 라인업 등록 시도]
부원 카드 표시 흐름:

  status = NORMAL
    → 일반 표시, 등록 가능

  status = CAUTION
    → [주의 -N] 노란 뱃지 표시, 등록 가능

  status = SUSPEND_1A / 1B (suspension_count > 0)
    → [출전 정지 잔여 N경기] 빨간 뱃지 표시
    → 등록 버튼 비활성 (회색)
    → 클릭 시 토스트: "출전 정지 상태입니다. 잔여 N경기"
    → 관리자가 [예외 등록] 클릭 시 사유 입력 모달 → 등록 가능
       (예외 등록은 ConductRecord에 자동 기록되어 감사 추적)

  status = EXPELLED
    → [퇴출] 회색 뱃지 표시
    → 등록 버튼 완전 차단 (예외 등록 옵션 없음)
    → 관리자 복권 후에만 등록 가능
```

### 1-4. 화면별 단계 표시 규칙

| 화면 | 표시 방식 |
|---|---|
| S05 출결현황(부원) | 블록 3 카드에 현재 단계 + 잔여 정지 경기 수 표시 |
| A02 부원 관리 | 부원 목록에 단계 컬럼 추가, 단계별 정렬 가능 |
| A07 라인업 등록 | 부원 카드에 단계 뱃지 + 등록 가능 여부 |
| A14 상벌점 관리 | 부원별 현재 단계 + 단계 변경 이력 |
| 마이페이지(부원) | 단계 안내 문구 + 만회 안내 ("상점 +N으로 NORMAL 복귀") |

### 1-5. 자동 알림 정책 — **없음**

- 임계치 도달 시 푸시·인앱·이메일 알림 **모두 미발송**
- 단계 변경은 화면에 조용히 반영
- 부원이 출결현황 화면에 진입했을 때 본인 단계 확인하는 방식 (Pull)
- 관리자도 단계 변경 알림 받지 않음 (A14에서 직접 확인)

### 1-6. 시즌 리셋 정책 — **없음**

- 상벌점은 **시즌이 바뀌어도 리셋되지 않음**
- 영구 누적 (부원이 졸업·탈퇴할 때까지 유지)
- ConductRecord에서 `season_id` 필드는 **선택값**으로 변경 (참고용 태깅만, 집계는 시즌 무관)
- 화면 표시: "상점 누계 +N (영구)", "벌점 누계 −N (영구)"
- 비교: 포인트(Point)는 시즌 단위 리셋 유지 → 두 축의 운영 사이클 차이 명확화

| 축 | 시즌 리셋 |
|---|---|
| 포인트 (Point) | 시즌 단위 리셋 |
| **상벌점 (Conduct)** | **리셋 없음, 영구 누적** |

## 2. 데이터 모델 변경 (v2.3 → v2.4)

### 2-1. User 엔티티 — 상벌점 상태 캐시 추가

| 필드 | 타입 | 비고 |
|---|---|---|
| conduct_total | int | 상벌점 합계 캐시 (성능용) |

## 3. 단계 표시 UI 명세

### 3-1. 단계 뱃지 색상 코드

| 단계 | 라벨 | 색상 | 사용처 |
|---|---|---|---|
| NORMAL | 표시 없음 | — | — |

| SUSPEND_1B | `출전정지 −N` | 빨간색 (#ef4444 계열) + 잔여 경기 수 | 라인업·관리 화면 |
| EXPELLED | `퇴출` | 검정색 (#111827 계열) | 라인업·관리 화면 |

### 3-2. 부원 출결현황(S05) — 블록 3 갱신

```
┌──────────────────────────────────────┐
│ 블록 3. 상벌점 (학교) — 영구 누적     │
│  ──────────────────────────────────  │
│  상점 +12   벌점 −27   합 −15        │
│                                       │
│  [상태] 출전 정지 (잔여 1경기)        │
│  ※ 합계 −15 이상 도달로 자동 부여     │
│                                       │
│  [상점 +14를 받으면 CAUTION 복귀]    │
│  [상점 +15를 받으면 NORMAL 복귀]     │
│  ※ 잔여 정지 경기는 별도 소진 필요   │
│                                       │
│  [이력 및 사유 보기]                  │
└──────────────────────────────────────┘
```

만회 안내 문구를 명시해 부원이 상점 획득 동기를 갖도록 한다.
